## Background

Sweden is the launch market. Mobile `i18n.ts` exists but is partially wired. This wave finalizes the en + sv baseline; longer tail (Home / Wardrobe / Plan / Insights deep copy) is deferred to v1.0.1.

## Files touched

### New
- `mobile/src/i18n/locales/en.ts` — append-only structure mirroring web's hierarchy.
- `mobile/src/i18n/locales/sv.ts` — Swedish translations.

### Modified
- `mobile/src/lib/i18n.ts` — load `en` + `sv` files; device locale via `expo-localization`.
- 12 screens listed above — `tr('settings.foo')` swaps for hardcoded strings.

## Pattern reference

Web `src/i18n/locales/en.ts` + `sv.ts` are the source of strings — copy keys + values directly. Append-only convention applies (never reorganize existing keys — that breaks future agents).
